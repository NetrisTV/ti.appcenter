  • MSInitializationPriority

    The priority by which the modules are initialized. MSPriorityMax is reserved for only 1 module and this needs to be Crashes. Crashes needs to be initialized first to catch crashes in our other SDK Modules (which will hopefully never happen) and to avoid losing any log at crash time.
